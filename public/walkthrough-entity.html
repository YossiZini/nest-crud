Step 3: The Entity
item.entity.ts

An entity defines the shape of a database table. Each instance of the class represents a row. TypeORM uses decorators on the class to generate the SQL schema — no raw SQL or separate migration file needed for development.

src/items/item.entity.ts
import { Entity, PrimaryGeneratedColumn, Column } from 'typeorm';

@Entity()
export class Item {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column({ default: '' })
  description: string;
}

Line-by-Line Breakdown

1

@Entity()

@Entity()
export class Item {

The @Entity() decorator marks this class as a database table. By default, the table name is the lowercase version of the class name: item. You can customize it: @Entity('products') would create a products table instead.

When synchronize: true is set in your TypeORM config (as we have in AppModule), Nest will automatically create this table in PostgreSQL when the app starts. If the table already exists, it updates the schema to match.

Express equivalent: If you used Sequelize, this is like sequelize.define('Item', { ... }). If you used Mongoose, it's like new Schema({ ... }). TypeORM uses a class-based approach with decorators instead of a config object.
2

@PrimaryGeneratedColumn()

@PrimaryGeneratedColumn()
id: number;

This creates an auto-incrementing integer primary key column. The SQL generated is:

-- Generated SQL
id SERIAL PRIMARY KEY

You can also use @PrimaryGeneratedColumn('uuid') to get UUID primary keys instead of integers. The id property is automatically populated after saving a new item.

3

@Column() — required field

@Column()
name: string;

A basic column with no options. TypeORM infers the SQL type from the TypeScript type:

TypeScript typePostgreSQL type
stringvarchar(255)
numberinteger
booleanboolean
Datetimestamp

Since there's no nullable: true option, this column is NOT NULL by default. Trying to save an Item without a name will throw a database error.

4

@Column() — with default value

@Column({ default: '' })
description: string;

The default: '' option sets a default value at the database level. If you save an Item without providing a description, PostgreSQL will use an empty string. The generated SQL is:

-- Generated SQL
description varchar(255) NOT NULL DEFAULT ''

Other useful column options:

  • { nullable: true } — allows NULL values
  • { unique: true } — adds a UNIQUE constraint
  • { length: 500 } — sets varchar length
  • { type: 'text' } — override the SQL type

What This Creates in PostgreSQL

When the app starts with synchronize: true, TypeORM runs the equivalent of:

CREATE TABLE item (
  id SERIAL PRIMARY KEY,
  name varchar(255) NOT NULL,
  description varchar(255) NOT NULL DEFAULT ''
);

How the Entity Connects to the Rest of the App

The entity alone doesn't do anything. It needs to be wired into the module system:

  1. AppModule → entities: [Item] in TypeOrmModule.forRoot() tells TypeORM about this entity
  2. ItemsModule → TypeOrmModule.forFeature([Item]) makes the Repository<Item> available for injection
  3. ItemsService → @InjectRepository(Item) receives the repository to run queries
Key takeaway: The entity is your single source of truth for the database schema. Change a property here, and TypeORM updates the table. The same class is also used as the return type for your API responses — what you define here is what your API clients see.
