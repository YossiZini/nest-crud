Step 1: The Entry Point
main.ts

Every Nest application starts with a single file: main.ts. This is the equivalent of your Express index.js where you call app.listen(). Let's walk through every line.

src/main.ts
import { NestFactory } from '@nestjs/core';
import { NestExpressApplication } from '@nestjs/platform-express';
import { ValidationPipe } from '@nestjs/common';
import { AppModule } from './app.module.js';

async function bootstrap() {
  const app = await NestFactory.create<NestExpressApplication>(AppModule);
  app.useGlobalPipes(new ValidationPipe({ whitelist: true }));
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();

Line-by-Line Breakdown

1

Imports

import { NestFactory } from '@nestjs/core';
import { NestExpressApplication } from '@nestjs/platform-express';

NestFactory is the main class that boots up a Nest app. It's like calling express() but with the entire DI container, module system, and middleware pipeline baked in.

NestExpressApplication is a type that tells TypeScript: "this app uses Express underneath." This gives you type-safe access to Express-specific methods like app.set(), app.engine(), etc.

Express equivalent: const app = express(); — that single line is what Nest replaces with the factory pattern.
2

The ValidationPipe import

import { ValidationPipe } from '@nestjs/common';

ValidationPipe is a built-in Nest pipe that automatically validates request bodies against your DTO classes. In Express, you'd write your own validation middleware or use something like express-validator. Nest gives you this out of the box.

3

Importing the root module

import { AppModule } from './app.module.js';

This is where Nest diverges from Express. Instead of manually registering routes, middleware, and database connections in your entry file, you import a single root module that declares the entire app structure. Everything — routes, services, database config — is organized inside this module tree.

4

Creating the app

const app = await NestFactory.create<NestExpressApplication>(AppModule);

This single line does a lot:

  1. Creates an Express instance internally
  2. Scans AppModule and all its imported modules
  3. Instantiates all controllers and services via the DI container
  4. Registers all routes defined by controller decorators
  5. Sets up the Express middleware pipeline

The <NestExpressApplication> generic explicitly tells Nest to use Express as the HTTP platform. This is the default, but being explicit makes it clear to anyone reading the code.

In Express: You'd need 20+ lines to wire up the same thing — create the app, register body-parser, set up each router, initialize database connections, etc. Nest does all of this from the module tree.
5

Global validation

app.useGlobalPipes(new ValidationPipe({ whitelist: true }));

This registers the ValidationPipe globally, meaning every incoming request will be validated against the corresponding DTO before it reaches your controller method.

The whitelist: true option strips any properties that are not defined in the DTO. If someone sends { "name": "Widget", "isAdmin": true } but your DTO only has name and description, the isAdmin field is silently removed. This prevents mass assignment attacks.

Express equivalent: You'd need a middleware like:
app.use((req, res, next) => { /* validate req.body against schema... */ })
With Nest, you just declare validation rules on your DTO class and it happens automatically.
6

Starting the server

await app.listen(process.env.PORT ?? 3000);

This calls Express's app.listen() under the hood. The ?? operator provides a default port of 3000 if no PORT environment variable is set. Exactly the same as in Express.

7

The bootstrap pattern

async function bootstrap() { ... }
bootstrap();

Nest wraps the startup in an async function called bootstrap because NestFactory.create() and app.listen() are both async operations. You can't use top-level await in all environments, so this pattern ensures compatibility. You'll see this in every Nest project.

What Happens at Startup

When you run npm run start:dev, here's the sequence:

  1. bootstrap() is called
  2. NestFactory.create() scans AppModule
  3. AppModule imports TypeOrmModule → Nest connects to PostgreSQL
  4. AppModule imports ServeStaticModule → Nest serves the public/ folder
  5. AppModule imports ItemsModule → Nest scans its controllers/providers
  6. ItemsController is instantiated with ItemsService injected
  7. ItemsService is instantiated with the TypeORM Repository<Item> injected
  8. All route decorators (@Get, @Post, etc.) are registered on the Express router
  9. ValidationPipe is registered globally
  10. Express starts listening on port 3000
Key takeaway: main.ts is intentionally minimal. All the real configuration lives in the module tree. This file just boots the engine and applies global settings.
